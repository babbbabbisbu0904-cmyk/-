# 引き継ぎノート デジタル化プロトタイプ（1事業所版）

Google スプレッドシート + Google Apps Script（GAS）だけで動く、引き継ぎノートのWebアプリです。
新しいツールを追加せず、いま使っているGoogle Workspaceの範囲内で動きます。

## できること

- 引き継ぎ事項をWeb画面から入力・閲覧（紙のノート不要）
- 出勤するスタッフに「前回出勤日以降に追加された、まだ自分が読んでいない引き継ぎ」を自動抽出して強制表示
- 「確認しました」ボタンで既読を記録（誰がいつ読んだか可視化）
- 新規登録時に該当建物のスタッフへ自動メール通知
- 出勤予定日の朝、未読が残っているスタッフへ自動リマインドメール
- 管理者は自分の管轄建物の引き継ぎと既読状況を一覧で確認可能

## セットアップ手順

### 1. スプレッドシートを新規作成

Google スプレッドシートを新規作成し、名前を「引き継ぎノート（プロトタイプ）」などにします。

### 2. Apps Script エディタを開く

スプレッドシートのメニューから「拡張機能」→「Apps Script」を開きます。

### 3. コードを貼り付け

- デフォルトで存在する `コード.gs`（または `Code.gs`）の中身を全部消し、このフォルダの `Code.gs` の内容を貼り付けます。
- 左側の「＋」→「HTML」で新規ファイルを作成し、ファイル名を **`Index`**（拡張子は不要）にします。このフォルダの `Index.html` の内容を貼り付けます。
- 保存します（Ctrl+S / Cmd+S）。

### 4. 初期セットアップを実行

スプレッドシートの画面に戻り、リロードすると「引き継ぎシステム」という独自メニューが追加されます。

1. 「① 初期セットアップ（シート作成）」を実行 → 初回は権限の承認画面が出るので許可してください。
2. 「職員マスタ」「職員建物マッピング」「建物マスタ」「シフト表」の4シートに実際のデータを入力します。
   - 職員マスタ: 氏名 / メールアドレス / 建物(未使用) / 管理者フラグ（TRUE/FALSE）
   - 職員建物マッピング: 氏名 / メールアドレス / 建物（1人が複数建物に出勤する場合は行を複数追加）
   - 建物マスタ: 建物名（対象事業所内の建物を1行ずつ）
   - シフト表: 日付 / 建物 / 氏名 / メールアドレス（誰がいつどの建物に出勤予定かの簡易シフト）

すぐに動作を試したい場合は「② サンプルデータを入力（テスト用）」を実行し、入力された職員マスタ・職員建物マッピングのメールアドレスを、実際にログインテストするご自身のGoogleアカウントに書き換えてください。

### 5. Webアプリとして公開

Apps Script エディタの右上「デプロイ」→「新しいデプロイ」から以下の設定で公開します。

- 種類: **ウェブアプリ**
- 実行するユーザー: **アクセスしているユーザー**
- アクセスできるユーザー: 組織内の全員（テスト段階では「自分のみ」でも可）

発行されたURLにアクセスし、画面が表示されることを確認します。
（初回アクセス時に「このアプリは確認されていません」という警告が出ることがありますが、社内向けの自作アプリなので「詳細」→「（プロジェクト名）に移動」で進めます）

### 6. 毎朝のリマインドを有効化

メニューから「③ 毎朝のリマインドを設定」を実行すると、毎日朝6時に「本日出勤予定かつ未読が残っているスタッフ」へ自動でリマインドメールが届くようになります。

## 動作確認のポイント

- 職員マスタに登録されていないGoogleアカウントでアクセスすると「職員マスタに登録されていないアカウントです」と表示されます（未登録者の閲覧を防ぐ仕組みです）。
- シフト表に自分の出勤日を入れた状態で、その日より後に別アカウントで引き継ぎを登録すると、「未読の引き継ぎ」欄にその内容が表示されます。
- 「確認しました」を押すと、既読ログシートに記録され、管理者ビューの「既読/未読」にも反映されます。
- 管理者フラグをTRUEにした職員でログインすると、画面下部に管理者ビューが表示されます。

## 複数建物に出勤するスタッフへの対応

1人のスタッフが複数建物に出勤する場合は、「職員建物マッピング」シートに、同じ氏名・メールアドレスで建物だけを変えた行を追加してください（例: 佐藤花子が第一ホームと第二ホームの両方に出勤するなら2行登録）。

- 未読判定は建物ごとに「その建物での前回出勤日」を基準に行われます（シフト表の日付×建物×メールアドレスで判定）。
- Webアプリを開くと、本日その人がシフト表で出勤予定になっている建物が自動で選ばれます。該当日のシフトが無ければ直近に出勤した建物、それも無ければマッピング上の先頭の建物が選ばれます。
- 複数建物に所属している場合は画面上部にプルダウンが表示され、手動で表示する建物を切り替えられます。
- 新規の引き継ぎ登録・既読確認・履歴表示は、すべて選択中の建物に対して行われます。

### 既存の「1人1建物」版からの移行

すでに職員マスタの「建物」列だけでスタッフを管理していた場合は、スプレッドシートのメニューから「引き継ぎシステム」→「【移行】職員マスタの建物列→職員建物マッピングへ」を実行してください。職員マスタの内容が自動的に職員建物マッピングシートにコピーされます。実行後、複数建物に出勤するスタッフの行を職員建物マッピングシートに追加すれば完了です（職員マスタの「建物」列自体はもう使われないので、残しておいても削除しても問題ありません）。

## 複数事業所への拡張について

5事業所13建物に展開する場合は、たとえば「建物」列を「事業所名＋建物名」の組み合わせにし、職員マスタに「管轄事業所（管理者用、複数可）」の列を追加すれば、同じ仕組みのまま拡張できます。事業所が増えてスタッフ数や通知量が多くなってきた場合は、kintoneなど専用の業務システムへの移行も選択肢になります。
